Orderflow Java Binding

Artifact: io.github.gregorian-09:orderflow-java-binding

Production-oriented Java (JNA) wrapper for the stable Orderflow C ABI, focused on analytics snapshots, signal streams, and external ingest.

Primary Packages

  • com.orderflow.bindings: SDK API surface.
  • com.orderflow.examples: runnable examples.

Runtime Lifecycle

  1. Create OrderflowEngine with EngineConfig.
  2. Call start().
  3. Subscribe streams (optional listeners).
  4. Drive with pollOnce(...) and/or external ingest.
  5. Read snapshots/metrics.
  6. Call stop() and close() (or try-with-resources).

Main Public Types

  • OrderflowEngine: lifecycle, subscriptions, polling, ingest, snapshots.
  • EngineConfig: immutable runtime configuration.
  • Symbol: venue + instrument + depth descriptor.
  • OrderflowEvent: callback envelope.
  • EventListener: callback interface.
  • constants: StreamKind, Side, BookAction, DataQualityFlags.

Snapshot Contracts

The following methods return JSON payload strings: bookSnapshot, analyticsSnapshot, signalSnapshot, and metricsJson.

External Ingest Contracts

External event flow is configured with configureExternalFeed(staleAfterMs, enforceSequence) and fed through ingestTrade(...) and ingestBook(...). Health state can be updated via setExternalReconnecting(...) and externalHealthTick().

Native Library Resolution

  1. Explicit constructor path.
  2. ORDERFLOW_LIBRARY_PATH environment variable.
  3. Default debug target path under target/debug.

Operational Notes

  • Callbacks are dispatched during pollOnce(...) and ingest paths.
  • Callback handlers should remain non-blocking.
  • Validate side/action/stream constants before ingest/subscribe.
