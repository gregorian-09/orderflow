Orderflow Java Binding

Artifact: io.github.gregorian-09:orderflow-java-binding

Java (JNA) wrapper for the stable Orderflow C ABI, focused on orderflow analytics, feed-health supervision, and production runtime integration.

Primary APIs

  • OrderflowEngine: lifecycle, subscription, ingest, snapshots.
  • EngineConfig: immutable runtime configuration.
  • Symbol: venue + instrument + depth descriptor.
  • OrderflowEvent: callback event envelope.

Runtime Model

  1. Create engine with configuration and optional native library path.
  2. Subscribe streams and optionally attach callback listeners.
  3. Drive runtime using polling and/or external ingest.
  4. Read JSON snapshots (book, analytics, signal, metrics).

Native Library Lookup

  1. Explicit constructor path.
  2. ORDERFLOW_LIBRARY_PATH environment variable.
  3. Default debug target path under target/debug.
